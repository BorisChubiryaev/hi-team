# Отпуска сотрудников — дизайн

Дата: 2026-07-20. Статус: утверждён.

## Задача

Сотрудник, уходя в отпуск, отмечает это в своём последнем отчёте — и на время
отпуска система не требует от него отчётов: он не попадает в «не сдали», не
получает напоминания бота и не портит аналитику дисциплины. Руководитель может
оформлять и править отпуска за других (болезнь, забыл отметить).

## Принятые решения

- **Срок — гибрид**: сотрудник выбирает 1–4 недели или «пока не вернусь»
  (открытый отпуск, длится до первого сданного отчёта).
- **Права**: сотрудник управляет своим отпуском через форму отчёта;
  LEAD/DIRECTOR — отпуском любого сотрудника в админке «Команда».
- **Аналитика**: недели отпуска исключаются из знаменателя дисциплины
  («сдали 3 из 4», а не «3 из 5»).
- **Telegram-бот в v1 — только учёт**: не напоминает отпускникам, показывает их
  отдельной строкой в групповом ростере. Оформить отпуск через бота нельзя.

## Модель данных

Новая модель в `prisma/schema.prisma` (+ relation `vacations Vacation[]` в `User`):

```prisma
model Vacation {
  id          String    @id @default(cuid())
  userId      String
  startDate   DateTime  @db.Date  // понедельник первой недели отпуска
  endDate     DateTime? @db.Date  // понедельник ПОСЛЕДНЕЙ недели (включительно); null = «пока не вернусь»
  createdById String?   // кто оформил (сам сотрудник или руководитель)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId, startDate])
}
```

Границы — понедельники недель, в той же конвенции, что `Week.startDate`
(см. `lib/weeks.ts`). Гранулярность — целая неделя.

**Инвариант**: у пользователя не больше одного «актуального» отпуска
(`endDate = null` или `endDate >= текущая неделя`). Создание нового актуального
отпуска заменяет прежний актуальный. Завершённые (исторические) отпуска не
трогаются — они нужны аналитике.

## Логика: `lib/vacations.ts`

Новый модуль, единственная точка работы с отпусками:

- `getVacationingUserIds(weekStart)` — id всех, у кого неделя `weekStart`
  покрыта отпуском (`startDate <= weekStart` и (`endDate = null` или
  `endDate >= weekStart`)).
- `getActiveVacation(userId)` — актуальный отпуск пользователя (для формы
  отчёта и админки).
- `setVacation(userId, startWeekIso, weeks | null, createdById)` — создать/
  заменить актуальный отпуск. `weeks = null` → открытый.
- `cancelVacation(userId)` — удалить актуальный отпуск (снял галочку,
  админ отменил).
- `closeVacationOnReport(userId, weekStartIso)` — автовозврат: вызывается при
  сохранении **непустого** отчёта за неделю X. Если отпуск покрывает X:
  `endDate = X − 1 неделя`; если отпуск начинался в X — удалить целиком.

## Статус недели: `lib/reminders.ts`

`getWeekStatus()` возвращает третью корзину `vacation: UserLite[]`:

- `vacation` — активные пишущие роли, чья текущая неделя покрыта отпуском;
- `missing` — как раньше, минус отпускники.

Следствия без доп. правок логики: личные напоминания не уходят отпускникам;
«✅ все сдали» срабатывает, когда сдали все не-отпускники. В `sendGroupRoster`
добавляется строка `🏖 В отпуске (N): …`, когда список непуст.

## Форма отчёта: `components/ReportForm.tsx`, `lib/reports.ts`

- Внизу формы блок: галочка «Со следующей недели я в отпуске» + селект срока
  (1 / 2 / 3 / 4 недели / «пока не вернусь»). 
- Предзаполнение: если у пользователя есть актуальный отпуск, начинающийся
  после недели отчёта, галочка стоит и срок выбран.
- Сохранение (`saveUserReport` получает новый опциональный параметр
  `vacation?: { weeks: number | null } | null`):
  - галочка стоит → `setVacation(userId, неделя отчёта + 1, weeks, userId)`;
  - галочка была и снята → `cancelVacation(userId)`;
  - параметр не передан (Telegram-бот) → отпуска не трогаем.
- Всегда при сохранении непустого отчёта: `closeVacationOnReport(userId,
  неделя отчёта)` — автовозврат и закрытие открытых отпусков.
- Отчёт задним числом (за прошлую неделю): отпуск стартует с недели, следующей
  за неделей **отчёта** — может оказаться текущей неделей, это корректно.

## Дашборд и аналитика

- `app/dashboard/page.tsx`: в карточке недели строка «В отпуске: Имя, Имя»
  (по аналогии с «Не сдали»), отпускники исключены из «не сдали»; счётчик
  «сдали N/M» — M без отпускников. Баннер «Ещё не сдали…» также без них.
- `lib/analytics.ts`: одним запросом загружаются отпуска, пересекающие
  анализируемый период; для каждой недели отпускники убираются из знаменателя
  дисциплины (и из числителя, если вдруг сдали — сдал в отпуске = не считаем
  неделю вовсе).

## Админка «Команда»

В строке сотрудника (`components/AdminUserRow.tsx` + server actions в
`app/admin/`): бейдж текущего/запланированного отпуска («в отпуске до 10 авг»,
«отпуск с 27 июля, открытый»), кнопки: оформить отпуск (начало = ближайшая
неделя на выбор: текущая/следующая; срок 1–4 недели или открытый), завершить
досрочно (= `cancelVacation`, если отпуск ещё не начался, или
`endDate = прошлая неделя`, если уже идёт). Доступно только `MANAGER_ROLES`.

## Мелкие штрихи

- AI-сводка недели (`lib/summary.ts`): в контекст промпта добавляется строка
  «В отпуске на этой неделе: …», чтобы модель не писала «X ничего не сдал».
- Экспорт недели `.md` / `.docx`: строка «В отпуске: …» рядом со списком
  сдавших/не сдавших.

## Крайние случаи

- Пустой отчёт (0 проектов) не считается сдачей и **не** вызывает автовозврат.
- Повторное сохранение отчёта галочку не дублирует: `setVacation` заменяет
  актуальный отпуск.
- Сотрудник может снять галочкой отпуск, оформленный админом (это его отпуск).
- DIRECTOR отчёты не пишет — в ростерах и дисциплине не участвует, отпуск для
  него в UI отчёта не предлагается (в админке — можно, но нигде не влияет).
- Неактивные (`active = false`) пользователи в отпусках не участвуют.

## Вне объёма v1

- Оформление отпуска через Telegram-бота (`/vacation`).
- Учёт отпусков в личном `ReviewPrep` и месячных сводках.
- Частичные недели (отпуск с середины недели).

## Проверка

Автотестов в проекте нет. Проверка через dev-сервер:

1. Отметить отпуск (2 недели) в отчёте → на следующей неделе сотрудник в
   строке «В отпуске», не в «не сдали», счётчик уменьшился.
2. Открытый отпуск → сдать отчёт → отпуск закрыт прошлой неделей.
3. Досрочный возврат: отчёт в середине фиксированного отпуска → усечение.
4. Админ оформил/завершил отпуск за другого.
5. Аналитика: неделя отпуска исключена из знаменателя дисциплины.

Плюс `npm run lint` и `npm run build`.
