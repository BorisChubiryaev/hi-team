# hi-team — еженедельные отчёты команды

Веб-приложение для еженедельных отчётов команды вместо таблицы в Confluence.
Команда входит по **email + паролю**, заполняет отчёт в структуре
**Проект → Сделано / Блокеры / Планы**, а встроенный ИИ (OpenRouter) делает
сводку недели для руководителя.

## Стек

- **Next.js 16** (App Router, TypeScript) + **Tailwind CSS v4**
- **Auth.js (NextAuth v5)** — вход по email + паролю (Credentials, JWT, bcrypt)
- **Prisma + Neon Postgres**
- **OpenRouter** — AI-суммаризация (серверный route, ключ не уходит в браузер)
- Хостинг — **Vercel**

## Локальный запуск

1. Установить зависимости:
   ```bash
   npm install
   ```
2. Создать `.env` из примера и заполнить:
   ```bash
   cp .env.example .env
   ```
   - `DATABASE_URL` — строка подключения Neon (раздел **Connect**, pooled).
   - `AUTH_SECRET` — сгенерировать: `npx auth secret`.
   - `OPENROUTER_API_KEY` — **новый** ключ OpenRouter (старый отозвать!).
   - `ALLOWED_EMAILS` — почты команды через запятую.
3. Применить схему и засеять реальные данные из Confluence:
   ```bash
   npm run db:push
   npm run db:seed
   ```
4. Запустить:
   ```bash
   npm run dev
   ```
   Открыть http://localhost:3000. При первом входе нажмите **«Первый вход? Задать
   пароль»**, укажите почту из `ALLOWED_EMAILS` и задайте пароль — дальше вход по
   email + паролю.

| --- | --- |
| `app/login` | Вход по email + паролю (+ первый вход) |
| `app/dashboard` | Таблица недели × сотрудники + AI-сводка, «кто не сдал», экспорт |
| `app/report` | Форма отчёта: выбор недели (текущая + 3 прошлые), черновик, автодополнение |
| `app/projects` | Проекты: статусы, история, AI-статус, переименование/слияние (LEAD) |
| `app/monthly` | AI-итоги месяца для отчётности наверх |
| `app/admin` | Команда (только LEAD): роли, доступ, allowlist, Telegram chat_id |
| `app/api/summary` | Генерация AI-сводки недели (OpenRouter) |
| `app/api/month-summary` | Генерация AI-итогов месяца |
| `app/api/projects/[id]/summary` | AI-статус проекта за последние недели |

| `app/api/cron/summary` | Cron: авто-сводка недели (пятница вечером) |
| `app/api/cron/remind` | Cron: напоминание не сдавшим отчёт (четверг) |
| `lib/auth.ts` | Конфиг Auth.js (Credentials + allowlist) |
| `lib/openrouter.ts` | Вызов OpenRouter + промпты (неделя, проект) |
| `lib/summary.ts` | Генерация сводки недели (общая для API и cron) |
| `lib/notify.ts` | Уведомления: Telegram и/или webhook |

| `prisma/seed.ts` | Реальные отчёты команды (июнь 2026) |
| `prisma/backfill-projects.ts` | Бэкфилл: создаёт `Project` из имён в старых отчётах |

## Роли и доступ

- **Роли**: `LEAD` (руководитель) и `MEMBER`. Лид управляет статусами,
  переименованием и слиянием проектов, а также командой на странице
  `/admin`: роли, деактивация ушедших (не логинятся и не показываются на
  дашборде), Telegram chat_id для личных напоминаний.
- **Allowlist** входа хранится в БД (таблица `AllowedEmail`, управляется из
  админки). Пока таблица пуста, действует старый fallback — env
  `ALLOWED_EMAILS`.
- Первый лид назначается сидом (Борис) или вручную:
  `UPDATE "User" SET role = 'LEAD' WHERE email = '...'`.

## Проекты и AI

- **Проекты** — отдельная сущность (`Project`): статус (Активен / На паузе /
  Завершён), история упоминаний по неделям, AI-статус за последние недели для
  отчётности наверх. Имена в форме отчёта автодополняются из активных проектов.
- Если в базе уже есть отчёты, созданные до появления проектов, один раз
  выполните `npm run db:backfill-projects` — строки отчётов сгруппируются по
  имени и привяжутся к проектам.
- **Черновик отчёта**: если отчёт за выбранную неделю ещё не создан, форма
  предзаполняется из последнего отчёта (планы → заготовка «Сделано», блокеры
  переносятся). Заполнить можно текущую и три прошлые недели.
- **Сводка недели** учитывает блокеры прошлых двух недель и помечает
  «висящие» блокеры.
- **Итоги месяца** (`/monthly`): агрегация недельных сводок и отчётов за
  месяц + экспорт в Markdown вместе со всеми неделями.
- **Гигиена проектов**: лид может переименовать проект или слить дубль в
  основной (упоминания перепривязываются). Активные проекты, не упоминавшиеся
  2+ недели, помечаются бейджем «нет движения».

## Cron и уведомления

`vercel.json` настраивает два задания Vercel Cron (роуты защищены
`CRON_SECRET`, Vercel подставляет его в `Authorization: Bearer ...`):

- `/api/cron/remind` — четверг 07:00 UTC: напоминание тем, кто не сдал отчёт.
- `/api/cron/summary` — пятница 15:00 UTC: авто-генерация AI-сводки недели.

Уведомления шлются в настроенные каналы: Telegram (`TELEGRAM_BOT_TOKEN` +
`TELEGRAM_CHAT_ID`) и/или webhook `NOTIFY_WEBHOOK_URL` (POST `{"text": ...}`,
совместим со Slack/Mattermost). Если каналы не настроены, cron отработает без
отправки. Напоминание дополнительно уходит **лично** тем не сдавшим, у кого в
админке заполнен Telegram chat_id (нужен тот же бот; chat_id сотрудник может
узнать, написав боту, — id придёт в `getUpdates`).

## Деплой на Vercel

1. Запушить репозиторий на GitHub и импортировать в Vercel.
2. В **Settings → Environment Variables** добавить все переменные из `.env`
   (`APP_URL` — на адрес деплоя, `CRON_SECRET` — случайная строка).
3. Применить схему к Neon один раз: локально с прод-`DATABASE_URL` выполнить
   `npm run db:push` и при необходимости `npm run db:seed`; для существующей
   базы — `npm run db:backfill-projects`.
4. Deploy. Auth.js определит домен автоматически (`trustHost: true`).

## Безопасность

- `OPENROUTER_API_KEY` и прочие секреты — только в переменных окружения, не в коде
  и не в git (`.env*` в `.gitignore`). Все вызовы ИИ — на сервере.
- Ключ OpenRouter, отправленный в переписку при постановке задачи, считается
  скомпрометированным — **отзовите его и выпустите новый**.
